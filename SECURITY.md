# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | :white_check_mark: |
| 1.0.x   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please follow these steps:

1. **Do NOT** open a public issue
2. Email security details to: [GitHub Security](https://github.com/ai-dev-2024/sonu/security/advisories/new)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Security Features

- **Offline Processing**: All audio processing happens locally
- **No Network Communication**: Application does not send data externally
- **Local Storage**: All data stored locally in JSON files
- **No Telemetry**: Zero tracking or analytics
- **Open Source**: Full code transparency

## Best Practices

- Keep dependencies updated
- Review code changes carefully
- Test security-critical features
- Follow secure coding practices

## Response Time

We aim to respond to security reports within 48 hours and provide updates within 7 days.

Thank you for helping keep SONU secure!
